Slide 3 — Lifecycle

Session: build, init, run, tear down

__init__(plan)
Validate the Mount Plan. An orchestrator and a context manager are required — missing either raises at construction.
initialize()
Load and mount modules in this fixed order:
  1. orchestrator   required — hard fail on error
  2. context manager   required — hard fail
  3. providers   soft fail per module (logged, session continues)
  4. tools   soft fail per module
  5. hooks   soft fail per module
  6. on_session_ready() callbacks across all modules — the one phase that sees the fully composed coordinator
execute(prompt)
Emit session:start (or session:resume). Delegate to the orchestrator. Track state: Running → Completed / Failed / Cancelled.
cleanup()
Emit session:end. Run registered cleanup functions in reverse order of registration. Errors are logged, never re-raised.
async with AmplifierSession(mount_plan) as session:
    response = await session.execute("hello")
# exiting the context calls session.cleanup() automatically
Slide 4 — Modules

Five protocols. That's the public surface.

A class with the right methods is the protocol. No base class to inherit, no decorator, no registry. (In Python, this is called structural typing.)

Provider Talks to an LLM backend (Anthropic, OpenAI, Pharia, Ollama…). complete(messages, model=…)
Tool An agent capability — bash, filesystem, web, search, code… execute(input) → ToolResult
Orchestrator The turn-by-turn loop (basic, streaming, model router…). execute(prompt, context, providers, tools, hooks)
ContextManager Conversation memory; decides compaction policy. add_message(), get_messages_for_request()
Hook Event handler — observability or policy intercept. __call__(event, data) → HookResult

Every module exposes one entry point — async def mount(coordinator, config) — which registers it on the coordinator's noticeboard.

Slide 5 — The loop

The Orchestrator is just a module — and it does the work

Every decision in the loop is policy. Hooks fire around each step. To change agent behaviour, swap the orchestrator.

async def execute(prompt, context, providers, tools, hooks, coordinator):
    await context.add_message({"role": "user", "content": prompt})

    while True:
        provider = self._pick_provider(providers)          # ← policy
        model    = self._pick_model(prompt)                # ← policy

        messages = await context.get_messages_for_request(provider=provider)
        await hooks.emit("llm:request", {"messages": messages})

        response = await provider.complete(messages, model=model)
        await hooks.emit("llm:response", {"response": response})
        await context.add_message(response.assistant_message)

        if not response.tool_calls:
            return response.text

        for call in response.tool_calls:
            await hooks.emit("tool:pre", call)              # deny / ask_user can fire here
            tool   = coordinator.get("tools", name=call.name)
            result = await tool.execute(call.input)
            await hooks.emit("tool:post", {"result": result})
            await context.add_message(tool_result_message(result))
Slide 6 — Two layering primitives

Configuration is data. Policy is data. Neither needs the kernel to know.

Bundles — configuration as data

bundle.md  # frontmatter + body
   ↓  compose()        # merge bundles, later wins
   ↓  prepare()        # fetch module sources
   ↓  to_mount_plan()
Mount Plan (dict)

Per-tenant configuration is a YAML overlay. Per-environment configuration is another. compose() layers them. No code change is needed to add or remove modules — only the bundle changes.

Hooks — policy as data

A handler receives an event and returns one of five actions:

  • deny short-circuit; return a refusal
  • ask_user open an approval prompt
  • inject_context append text to the next turn
  • modify transform the payload, then pass on
  • continue default — pure observation

Approval gates, redaction, audit logging, cost ceilings, allow-lists — every cross-cutting concern lives here. The kernel ships none of them.

Slide 7 — Take this home
Five protocols.

Two layering primitives — compose for config, hooks for policy.

The kernel is roughly 2,600 lines you can read in an afternoon. Everything else is naming parts of the picture from slide 2.
